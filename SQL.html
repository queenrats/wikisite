Server Stacks, SQL, and Common Security Problems


Server Stacks

Servers on the internet usually have a variety of software installed on them.

While this (perhaps obviously) includes the operating system, it can also include a number of other things such as database software and even the services needed to host a website in the first place.

While having lots of different pieces of software in the server stack is great for functionality, it can also introduce security vulnerabilities.


SQL

SQL is a language used for programming databases. There are a number of database management systems that use SQL, including MySQL, Microsoft SQL, MariaDB, and PostgreSQL.

SQL queries are designed to be easily read, and use whole uppercase words. For example, "SELECT * FROM data" selects all data (the *) from the database table "data".

There are plenty of other types of commands, such as UPDATE, INSERT INTO, and CREATE DATABASE.

If a server is using a database, it can dynamically populate websites with information from the database. However, this can sometimes be taken advantage of.

SQL injection is a way of executing SQL commands on a server's database through a website that isn't secured properly.

SQL injection requires a user input that queries a database, and involves an attacker "tagging on" something to the query. This could be as simple as viewing the whole table, or deleting it entirely.

A common way that SQL injection is tested for in database search functions is by entering the string "'OR '1=1" into the input box.

This works because of the SQL modifier WHERE, which means a search only returns values where something equals a given value, in this case the given value is what the user put in.

The way the apostrophes are structured means that, as well as the intended WHERE clause, it also returns rows where '1=1' is true, which since the statement is true, means every row is returned.


Common Security Problems

SQL injection isn't the only common problem that inseecure websites can face.

There is another type of injection called Script Injection, which involves executing Javascript on a site that shouldn't allow it.

A common way script injection is tested for is by trying to run a Javascript "alert" function inside HTML script tags. If an alert box appears, then the site is vulnerable to script injection.

As a reminder, script tags look like this: <script>, and an example of correct syntax for the alert function is "alert("test")

Other common security problems include Cross-Site Request Forgery (CSRF), Security Misconfiguration, and Insecure direct Object References.
