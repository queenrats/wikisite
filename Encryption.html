Encryption, Ciphers, and RSA


Stream Ciphers

Stream ciphers are a way of encrypting data on a network. One way that stream ciphers can work is by performing an XOR calculation between the bits (zeros and ones) of data being transmitted and a “key stream.”

An XOR calculation is a type of binary calculation where the result is 1 only when one input is a 1, otherwise the result is 0.

Given the same input, this is reversible, meaning if two people have access to a key stream they can send each other encrypted data over an unsecured network.

RSA Encryption

RSA (an acronym of Rivest-Shamir-Adleman, the names of the creators) encryption is an asymmetric cryptographic algorithm, which means that, instead of one key being shared, there are two different keys.

The two types of key are a public key, which is shared with everyone and can be used to encrypt data, and a private key, which is kept secret by the keyholder and is used to decrypt any data encrypted with the public key.

This means that anyone can send encrypted messages to someone, but only the intended recipient can decrypt and see the data being sent.

RSA keys are usually generated using two large prime numbers using the following mathematical calculations:

  1. Choose two prime numbers p and q
  2. Calculate n, where n = p*q
  3. Calculate the totient φ(n) which is equal to (p - 1)*(q - 1)
  4. Choose an integer e such that the greatest common denominator (GCD) of φ(n) and e is 1.
  5. Finally, calculate d so that d*e mod(n) = 1, where mod is the modulus function (most calculators have a button for this)

Once you have all those numbers, (n, e) is your public key and d is your private key.


Here's how to encrypt and decrypt data:

  1. Let M be the message you want to encrypt (as an integer).
  2. Calculate Me mod(n), where the result is your ciphertext.
  3. To decrypt the ciphertext C, calculate Cd mod(n)

Something important to note is that the integer you use for your message has to be smaller than n, otherwise it won't work.
